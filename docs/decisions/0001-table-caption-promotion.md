# ADR 0001 — 표 캡션 승격 정책 채택

- 상태: 승인 (2026-04-19)
- 작성자: Team B
- 근거 명세: `samples/incremental/05_table/05_table.md` (Team A 작성) front matter

## 배경

CommonMark / GFM 에는 "표 캡션" 을 위한 표준 문법이 없다. 즉 사용자는
표 직전에 자유 텍스트로 "표 1. 분기별 매출" 같은 줄을 적고 시각적으로만
캡션처럼 다룬다.

한/글 (HWPX) 은 표 노드(`<hp:tbl>`) 안에 명시적인 캡션 노드
(`<hp:caption>`) 를 가지며, 거기 들어간 단락은 "표캡션" 스타일을 받고
번호는 `<hp:autoNum num="N" numType="TABLE">` 으로 자동 매겨진다.

## 결정

마크다운 → 한/글 변환 시 **표 직전 단락이 정규식
`^(표|Table)\s+\d+\.\s*` 로 시작하면 표의 캡션으로 승격**한다.

- 접두사 (`표 1. `, `Table 3. ` 등) 는 파싱 후 제거하고, 그 뒤의 텍스트만
  캡션 본문으로 저장한다.
- 사용자가 적은 번호 값은 **무시**한다. 한/글이 문서 내 표 등장 순서대로
  1, 2, 3 … 을 자동 부여한다 (`<hp:autoNum>`).
- 직전 단락이 일반 단락(`role="paragraph"`) 일 때만 승격 대상. 헤딩/인용/
  코드 등은 승격하지 않는다.
- "직전" 의 정의: walker 가 평탄 Block 리스트를 순회할 때 표 Block 의
  바로 앞 인덱스. 마크다운 원본에서 빈 줄이 사이에 있어도 markdown-it 의
  토큰화 결과에서는 두 단락 사이에 빈 Block 이 들어가지 않으므로 인접
  여부는 인덱스로 판정.

## 대안과 기각 사유

### 대안 1 — 캡션 무시 (휴리스틱 미적용)
- 장점: 본문이 캡션으로 오인될 위험 0. 우리 변환기 철학("구조만 옮긴다")
  와 정합.
- 단점: 사용자가 한/글에서 매번 "이 단락을 표캡션 스타일로 바꾸기" 를
  수동으로 해야 한다. 흔한 패턴(`표 N. 제목`) 에 대한 자동화 가치를 잃음.

### 대안 2 — 명시적 마커 도입 (예: HTML 주석 `<!-- caption: ... -->`)
- 장점: 오인 위험 0. 의도가 명확.
- 단점: 마크다운 사용자가 우리 전용 문법을 학습해야 함. 다른 마크다운
  뷰어에서 캡션이 안 보임 (호환성 손실).

### 대안 3 — 채택 (A 명세 기반 정규식 승격)
- 장점: 한국어 학술 / 보고서 글쓰기에서 표준적인 캡션 표기 (`표 N.`,
  `Table N.`) 를 자동 인식. 마크다운 원본에서도 캡션처럼 읽힘
  (이중 가독성).
- 단점: 본문에서 "표 1. 의 결과를 보면 …" 같은 문장이 표 직전에 오면
  캡션으로 오인 가능. 다만 정규식 끝의 `\.\s*` 가 점 뒤 "공백 또는 끝"
  을 요구하므로 어느 정도 방어. 그래도 잠재 위험은 0 이 아님.

## 보호 장치

1. **승격 조건이 좁음**: 직전 Block 이 `role="paragraph"` 이고, 텍스트가
   정규식에 매치되며, 표가 같은 walker 순회 단계에 인접해 있어야만 발동.
2. **골든 회귀 픽스처**: `tests/golden/05_table_with_caption/` 가 있어
   캡션 동작이 의도치 않게 바뀌면 즉시 테스트가 깨진다.
3. **승격 결과 검증**: `mapsi.inspect` 로 변환 결과의 단락 시퀀스를 확인
   가능. 첫 행이 "표캡션" 스타일이면 승격됨, "본문" 으로 남으면 승격 안 됨.
4. **번호 보존 안 함의 명시**: 사용자가 적은 번호는 무시되고 한/글의
   `autoNum` 이 재부여한다. 이 차이는 `docs/c_handoff.md` 와 본 ADR 에
   기록되어 있다.

## 참고

- A 의 원 명세: `samples/incremental/05_table/05_table.md` 의 front matter
  "캡션 승격 정책" 절.
- 순서 목록의 정책과 비교: 순서 목록(03)은 원문 번호 텍스트를 *보존*
  하고 자동번호를 쓰지 않는다. 표 캡션은 반대로 원문 번호를 *제거*
  하고 한/글 autoNum 에 위임한다. 차이의 근거는 한/글의 표 자동번호
  메커니즘 존재 여부.
