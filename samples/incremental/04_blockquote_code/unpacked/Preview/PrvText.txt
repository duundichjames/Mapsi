마크다운의 blockquote(> 로 시작하는 블록) 와 code block(백틱 세 개로 감싼 블록) 에 대응하는 스타일을 지정함. hwpx 에는 이에 대응하는 내장 스타일이 없으므로, 마크다운과의 대응을 위해 사용자 스타일 두 개("인용", "코드") 를 정의함. 인용 스타일은 본문 대비 좌우 여백을 들여 본문과 시각적으로 구분하며, 코드 스타일은 한글 서체를 기본 본문 서체인 한컴바탕에서 한컴돋움으로, 영문 서체를 Times New Roman 에서 Courier New(맥에서도 설치되는 서체로 알고 있으나, 설치되지 않은 경우 Menlo로 설정)로 변경하여 코드 블록임을 드러냄. 두 스타일 모두 문단 번호/글머리표를 "없음" 으로 명시하여 다른 스타일의 번호 체계 영향을 받지 않도록 함.

인용 블록은 본문에서 분리되어 다른 화자나 문헌의 목소리를 드러낸다. 일반적으로 좌우 여백을 들이고 경계선이나 배경으로 본문과 구분한다.

def 스타일ID(이름):
    return 이름에서ID[이름]

def 문단만들기(텍스트, 역할, 문단ID):
    sid = 스타일ID(스타일매핑[역할])
    return f'<hp:p id="{문단ID}" styleIDRef="{sid}">...'

마크다운의 ordered list, 즉 "1. ", "\t1. ", "\t\t1. " 와 같이 항목에 순차적 번호를 붙여 나열하는 목록 형식에 대응하는 스타일을 지정함. 한/글(hwpx) 에는 이에 대응하는 내장 스타일이 없으므로, 마크다운과의 대응을 위해 사용자 스타일 세 개(번호1, 번호2, 번호3) 를 정의하여 각 중첩 깊이에 대응시킴. 한/글의 자동 번호매김 기능은 사용하지 않으며, 마크다운에서 그대로 전달된 번호 텍스트("1. ", "1. ", "1. " 등) 가 표시되도록 함. 

자동 번호매김을 배제하는 이유는 이 기능이 문단 단위로 적용되어 같은 번호 체계에 등록된 모든 단락이 하나의 카운터를 공유하기 때문임. 이 경우 앞서 정의한 개요 1 - 5 의 제목 번호
